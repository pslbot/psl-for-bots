# Code of Conduct

## Our Pledge

PSL Bot is a satirical software engineering project. We are committed to providing a welcoming, inclusive environment for contributors regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

**Critical Distinction**: This project satirizes tech industry behavior (framework wars, cargo cult adoption, tribalism) and looksmaxxing culture (superficial quantification of human value). It does NOT endorse the harmful aspects of looksmaxxing communities (misogyny, body dysmorphia promotion, toxicity).

## Our Standards

### Positive Behaviors

- **Respectful Critique**: Engage critically with the satirical premise, challenge assumptions constructively
- **Collaborative Spirit**: Help improve both the satire and the substance
- **Inclusive Language**: Welcome contributors of all backgrounds
- **Constructive Feedback**: Focus on code/content, not people
- **Graceful Disagreement**: Disagree about ideas, not identities
- **Empathy**: Remember there's a human behind every PR

### Unacceptable Behaviors

- **Harassment**: Intimidation, stalking, unwelcome sexual attention
- **Discrimination**: Racist, sexist, homophobic, transphobic, ableist comments
- **Personal Attacks**: Ad hominem arguments, insults
- **Trolling**: Bad-faith engagement, derailing discussions
- **Doxxing**: Publishing private information without consent
- **Spam**: Excessive self-promotion, off-topic posting
- **Misusing the Satire**: Using looksmaxxing terminology to harass people (it's for bots, not humans)

### Satirical Boundaries

**Acceptable**:
- Calling React "Gigachad-tier" (satirizing tech tribalism)
- Saying Webpack got "mogged" by Vite (competitive analysis)
- Using looksmaxxing terminology for bot comparison

**Unacceptable**:
- Applying PSL methodology to people
- Using looksmaxxing terminology to rate human attractiveness
- Promoting harmful looksmaxxing ideologies (misogyny, etc.)

**Guideline**: If it makes you uncomfortable to say about a person, don't say it about a bot either. Keep the satire punching up (at industry behavior), not down (at individuals).

## Scope

This Code of Conduct applies to:
- GitHub repository (issues, PRs, discussions)
- Project communication channels
- Public spaces representing the project
- Private communication when representing the project

## Enforcement

### Reporting

If you experience or witness unacceptable behavior:

1. **GitHub**: Report via GitHub's reporting tools
2. **Email**: conduct@psl-bot.dev (fictional - use GitHub in practice)
3. **Direct**: Contact maintainers directly

All reports will be reviewed and investigated promptly and fairly. Reporters' privacy and security will be respected.

### Consequences

Maintainers may take action including:

1. **Warning**: Private written warning explaining violation
2. **Temporary Ban**: Temporary suspension from project spaces
3. **Permanent Ban**: Permanent removal from project community

Factors considered:
- Severity of violation
- Pattern of behavior
- Context and intent
- Impact on community

### Appeal Process

If you believe enforcement action was unjust:
1. Email appeal to maintainers within 7 days
2. Explain why action was inappropriate
3. Maintainers will review and respond within 14 days

## Maintainer Responsibilities

Maintainers are responsible for:
- Clarifying standards of acceptable behavior
- Taking appropriate and fair corrective action
- Removing, editing, or rejecting contributions that violate this Code of Conduct
- Applying this Code of Conduct consistently

Maintainers who don't follow or enforce the Code of Conduct may face temporary or permanent repercussions.

## Philosophical Note on Satire

PSL Bot satirizes two things:

1. **Looksmaxxing Culture**: The absurdity of reducing human value to quantifiable metrics, the toxicity of online aesthetics communities, the pseudoscience of "objective" beauty.

2. **Tech Industry Tribalism**: Framework wars, cargo cult adoption ("FAANG uses it"), treating architectural preferences as moral imperatives, quantification fetishism.

**The Satire's Goal**: By applying looksmaxxing methodology to software (obviously absurd), we highlight how absurd it is to apply it to humans. By treating framework choice like beauty contests (obviously ridiculous), we highlight how ridiculous framework wars are.

**Use the satire responsibly**. Don't let irony poisoning make you actually believe software quality = human worth, or that calling webpack "Subhuman" is literally dehumanizing (it's a bundler, not a person).

**When in doubt**: Would this contribution make PSL Bot a better satirical critique of tech culture, or would it just be mean? Aim for the former.
